## 概述

`config.toml` 是 Codex CLI 的主配置文件，位于 `~/.codex/config.toml`。它允许你自定义模型选择、权限范围、自定义 API provider（中转服务）等。

## 文件位置

| 操作系统 | 路径 |
|---|---|
| macOS / Linux | `~/.codex/config.toml` |
| Windows | `C:\Users\<用户名>\.codex\config.toml` |

## 完整配置示例

```toml
# 模型配置
model = "o4-mini"
model_provider = "openai"

# 权限配置
approval_policy = "on-failure"
sandbox_mode = "workspace-write"

# 自定义 Provider（API 中转）
[model_providers.openai]
name = "OpenAI"
base_url = "https://api.openai.com/v1"
env_key = "OPENAI_API_KEY"

[model_providers.custom]
name = "Custom API"
base_url = "https://your-proxy-domain.com/v1"
env_key = "CUSTOM_API_KEY"
wire_api = "chat"

# MCP 服务器配置
[mcp_servers.filesystem]
command = "npx"
args = ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/dir"]
```

## 配置项详解

### model

指定 Codex 使用的模型：

```toml
model = "o4-mini"
```

常用模型：
- `o4-mini`：快速、经济（默认推荐）
- `o3`：更强推理能力
- `gpt-4.1`：通用模型
- `codex-1`：Codex 专用模型（如有权限）

### approval_policy

控制 Codex 执行命令前是否需要用户确认：

```toml
approval_policy = "on-failure"
```

可选值：
- `always`：每次操作都需要确认（最安全）
- `on-failure`：仅在可能失败时确认（默认，推荐）
- `never`：从不确认（全自动，风险最高）

### sandbox_mode

控制 Codex 的文件系统访问权限：

```toml
sandbox_mode = "workspace-write"
```

可选值：
- `read-only`：仅读取，不可写入
- `workspace-write`：仅可写入当前工作区（推荐）
- `full-access`：完整文件系统访问（危险）

### model_providers

定义 API provider，支持自定义中转服务：

```toml
[model_providers.custom]
name = "My Proxy"
base_url = "https://proxy.example.com/v1"
env_key = "CUSTOM_API_KEY"
wire_api = "chat"
```

- `name`：显示名称
- `base_url`：API 基础 URL（中转服务地址）
- `env_key`：读取 API Key 的环境变量名
- `wire_api`：API 协议类型（`chat` 或 `responses`）

### mcp_servers

配置 MCP（Model Context Protocol）服务器，扩展 Codex 的工具能力：

```toml
[mcp_servers.github]
command = "npx"
args = ["-y", "@modelcontextprotocol/server-github"]
env = { GITHUB_TOKEN = "ghp_xxxxxxxx" }
```

## 常用配置场景

### 场景一：使用 API 中转服务

```toml
model = "o4-mini"
model_provider = "custom"

[model_providers.custom]
name = "API Proxy"
base_url = "https://your-proxy.com/v1"
env_key = "PROXY_API_KEY"
wire_api = "chat"
```

设置环境变量：

```bash
export PROXY_API_KEY="your-proxy-key"
```

详见 [API 中转方案](/docs/codex/auth/api-proxy)。

### 场景二：最高安全性

```toml
model = "o4-mini"
approval_policy = "always"
sandbox_mode = "read-only"
```

每次操作都需要确认，且只读不写。

### 场景三：最大自由度

```toml
model = "o3"
approval_policy = "never"
sandbox_mode = "full-access"
```

<div class="alert alert-danger">
<strong>警告：</strong><code>full-access</code> + <code>never</code> 组合风险极高，Codex 可能执行危险操作（如删除文件）。仅在你完全信任的工作环境中使用。
</div>

### 场景四：配置 GitHub MCP

```toml
[mcp_servers.github]
command = "npx"
args = ["-y", "@modelcontextprotocol/server-github"]
env = { GITHUB_TOKEN = "ghp_xxxxxxxxxxxx" }
```

配置后，Codex 可以直接操作 GitHub（创建 PR、查看 Issue 等）。

## 配置优先级

Codex 读取配置的优先级从高到低：

1. **命令行参数**：`codex --model o3`
2. **环境变量**：`OPENAI_API_KEY`、`CODEX_MODEL` 等
3. **config.toml**：`~/.codex/config.toml`
4. **默认值**

## 常见问题

### Q: config.toml 修改后不生效？

重启 Codex CLI。配置文件在启动时读取。

### Q: TOML 格式报错？

TOML 对格式要求严格，常见错误：
- 字符串必须用双引号
- 表（table）标题用方括号 `[section]`
- 键值对用等号 `key = "value"`
- 注释用 `#`

### Q: 如何查看当前生效的配置？

```bash
codex --print-config
```
