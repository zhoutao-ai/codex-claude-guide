## 问题描述

在 Windows 上运行 Codex CLI 时，程序突然崩溃退出，终端显示退出码：

```
exit code: 3221225781
```

或将退出码转换为十六进制：`0xC0000005`。

## 错误含义

退出码 `3221225781`（十六进制 `0xC0000005`）是 Windows 的 **STATUS_ACCESS_VIOLATION**（内存访问违规）。这意味着程序试图读取或写入没有权限的内存地址。

常见触发场景：
- Codex CLI 与某些系统组件不兼容
- Node.js 版本与 Codex 不匹配
- 系统缺少必要的运行时依赖
- 杀毒软件拦截了 Codex 的操作

## 解决方案

### 方案一：更新到最新版本（首要操作）

```bash
npm update -g @openai/codex
```

或卸载后重新安装：

```bash
npm uninstall -g @openai/codex
npm install -g @openai/codex
```

### 方案二：检查 Node.js 版本

Codex CLI 要求 Node.js ≥ 22。版本过低可能导致兼容性问题：

```bash
node --version
```

如果低于 v22，使用 nvm-windows 升级：

```bash
nvm install 22
nvm use 22
```

然后重新安装 Codex：

```bash
npm install -g @openai/codex
```

### 方案三：检查系统架构

确认你使用的是 64 位版本的 Node.js：

```bash
node -p "process.arch"
```

应输出 `x64`。如果输出 `ia32`（32 位），需要卸载并安装 64 位版本的 Node.js。

### 方案四：排除杀毒软件干扰

部分杀毒软件（如 360、火绒、Windows Defender）可能拦截 Codex CLI 的文件操作：

1. **临时关闭杀毒软件**，测试 Codex 是否正常运行
2. 如果关闭后正常，将 Codex 和 Node.js 安装目录添加到杀毒软件的**白名单**中
3. 特别关注 `~/.codex/` 目录下的文件操作

### 方案五：使用官方安装脚本替代 npm

npm 安装的版本可能与系统不完全兼容。尝试使用官方安装脚本：

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://chatgpt.com/codex/install.ps1 | iex"
```

### 方案六：检查系统更新

确保 Windows 已安装最新更新：

1. 设置 → Windows 更新 → 检查更新
2. 安装所有待更新项
3. 特别注意 **Visual C++ Redistributable** 的更新

### 方案七：查看详细日志

如果以上方案都无效，查看 Codex 的详细日志：

```bash
codex --verbose
```

或在 `~/.codex/logs/` 目录下查看日志文件，寻找崩溃前的最后操作。

## 环境检查清单

| 检查项 | 要求 | 检查命令 |
|---|---|---|
| Node.js 版本 | ≥ 22 | `node --version` |
| 系统架构 | 64 位 | `node -p "process.arch"` |
| Codex 版本 | 最新 | `codex --version` |
| 内存 | ≥ 4GB | 系统信息 |
| 杀毒软件 | 已加白名单 | - |
| 系统更新 | 已安装最新 | Windows 更新 |

<div class="alert alert-danger">
<strong>如果所有方案都无效：</strong>可能是你的 Windows 版本存在深层兼容性问题。建议：1) 在 WSL2（Ubuntu）中运行 Codex CLI；2) 或使用 VS Code 扩展替代 CLI；3) 或在 [GitHub Issues](https://github.com/openai/codex/issues) 提交 bug 报告，附上退出码、系统版本、Node 版本等信息。
</div>
