## 问题描述

在 Claude.ai 注册时，页面显示验证码已发送：

```
Verification code sent to +XX XXXXXXXX
```

但手机始终收不到验证码短信。等待 10 分钟、30 分钟甚至更久，依然没有收到。

## 错误原因

### 原因一：区号不在服务区

Anthropic 的短信发送服务并非覆盖全球所有地区。以下区号的号码可能收不到验证码：

| 区号 | 地区 | 是否支持 |
|---|---|---|
| +86 | 中国大陆 | ✗ 不支持 |
| +852 | 中国香港 | ⚠ 不稳定 |
| +886 | 中国台湾 | ⚠ 不稳定 |
| +1 | 美国/加拿大 | ✓ 支持 |
| +44 | 英国 | ✓ 支持 |
| +61 | 澳大利亚 | ✓ 支持 |
| +49 | 德国 | ✓ 支持 |
| +33 | 法国 | ✓ 支持 |
| +81 | 日本 | ✓ 支持 |

### 原因二：运营商拦截

部分海外运营商可能将来自 Anthropic 的国际短信标记为垃圾短信或直接拦截。

### 原因三：漫游问题

如果你使用的是海外 SIM 卡但在中国大陆漫游，部分运营商可能不转发国际短信。

### 原因四：网络延迟

短信可能延迟到达，尤其是跨国短信。

## 解决方案

### 方案一：确认号码归属地

确保你的号码归属地在 Claude 支持的地区。推荐使用：
- **美国 (+1)**：最稳定
- **英国 (+44)**：稳定
- **澳大利亚 (+61)**：较稳定

避免使用 +86、+852、+886 等地区的号码。

### 方案二：检查手机设置

1. **关闭短信拦截**：部分手机有垃圾短信拦截功能，检查是否误拦
2. **检查短信中心号码**：确保 SIM 卡的短信中心号码正确
3. **重启手机**：有时重启可以解决短信接收问题
4. **切换网络**：从 Wi-Fi 切换到移动网络，或反之

### 方案三：使用其他号码

如果当前号码始终收不到：

1. 更换为另一个支持地区的号码
2. 确保新号码是实体线路（非 VoIP）
3. 等待 1 小时后使用新号码验证

### 方案四：使用语音验证（如支持）

部分情况下 Claude 可能提供语音电话验证选项。如果页面上有 "Try voice call" 选项，可以尝试。

### 方案五：检查漫游设置

如果你使用海外 SIM 卡在中国漫游：

1. 确保手机已开启**国际漫游**
2. 确保漫游设置中允许接收短信
3. 检查运营商是否支持在中国漫游时接收短信
4. 联系运营商客服确认短信漫游是否正常

## 各运营商漫游收短信能力

| 运营商 | 在中国漫游收短信 | 备注 |
|---|---|---|
| Ultra Mobile | ✓ 支持 | 需开启漫游 |
| Lycamobile | ✓ 支持 | 需开启漫游 |
| T-Mobile | ✓ 支持 | 需开启 Wi-Fi Calling |
| giffgaff | ⚠ 不稳定 | 部分用户反馈收不到 |
| AT&T 预付费 | ✗ 不支持 | 预付费卡不支持国际漫游 |

<div class="alert alert-warning">
<strong>购买前确认：</strong>购买海外 SIM 卡前，务必确认该卡支持在中国大陆漫游接收短信。部分预付费卡不支持国际漫游。
</div>

## 诊断步骤

1. **确认区号**：查看你的号码区号是否在支持列表中
2. **测试短信**：用另一个手机给该号码发短信，确认能正常接收
3. **检查漫游**：确认手机已开启国际漫游
4. **等待更久**：跨国短信有时延迟 5-15 分钟
5. **更换号码**：如果 30 分钟仍未收到，更换号码

## 常见问题

### Q: 页面提示发送成功但一直没收到怎么办？

等待 30 分钟。如果仍未收到，更换号码。不要频繁点击"重新发送"，否则可能触发 [临时封锁](/docs/claude/errors/temp-blocked)。

### Q: 之前能收到验证码，现在收不到了？

可能是运营商临时问题。等待几小时后重试，或更换号码。

### Q: 可以使用邮箱验证替代手机号验证吗？

Claude 注册时手机号验证是强制步骤，无法用邮箱替代。
