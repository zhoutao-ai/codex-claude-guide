## 问题描述

在 Claude.ai 注册时输入手机号后，收到提示：

```
phone_number_temp_blocked
```

或：

```
This phone number has been temporarily blocked due to too many attempts.
```

无法继续验证。

## 错误原因

Anthropic 的风控系统检测到同一号码或同一 IP 在短时间内发起了过多的验证请求。触发临时封锁的常见场景：

1. **频繁点击"发送验证码"**：短时间多次请求验证码
2. **多个号码失败**：在同一 IP 下尝试了多个不同的号码
3. **验证码输入错误**：多次输入错误的验证码
4. **IP 异常**：使用的 IP 被标记为高风险

临时封锁通常持续 **24 小时**，之后会自动解除。

## 解决方案

### 方案一：等待 24 小时（最简单）

如果号码本身没问题，只是因为尝试过多被锁：

1. 停止所有尝试
2. 等待 24 小时
3. 确保使用干净的环境（无痕模式 + 稳定 IP）
4. 重新验证

### 方案二：更换号码

如果不想等待 24 小时，或者号码本身也有问题：

1. 准备一个新的、未使用过的海外号码
2. 等待至少 1 小时（让 IP 的封锁冷却）
3. 使用新号码在干净环境下验证

### 方案三：更换环境 + 号码

如果 IP 也被标记了：

1. 更换 VPN/代理节点（切换到另一个干净的 IP）
2. 使用新的浏览器无痕窗口
3. 使用新的号码
4. 重新注册

### 方案四：使用 Google 账号注册

跳过手机号验证，直接使用 Google 账号注册：

1. 访问 [claude.ai](https://claude.ai)
2. 选择 "Continue with Google"
3. 授权登录
4. 如果不要求手机号验证，直接完成注册

## 预防措施

<div class="alert alert-warning">
<strong>避免触发临时封锁：</strong>
<ul>
<li>每次验证码请求之间至少等待 60 秒</li>
<li>不要在同一 IP 下连续尝试超过 3 个不同号码</li>
<li>验证码收到后仔细核对再输入，不要快速试错</li>
<li>如果第一次验证失败，不要立即重试，等待 1 小时</li>
<li>使用稳定的 IP，不要频繁切换节点</li>
</ul>
</div>

## 封锁时间线

| 触发条件 | 封锁时长 | 影响范围 |
|---|---|---|
| 短时间多次请求验证码 | 1-6 小时 | 号码 + IP |
| 验证码多次输入错误 | 1-2 小时 | 号码 |
| 多个号码验证失败 | 12-24 小时 | IP |
| 疑似机器人行为 | 24-72 小时 | IP + 浏览器指纹 |

## 常见问题

### Q: 24 小时后还是不行怎么办？

可能封锁时间延长了，或者号码/IP 被永久标记。更换号码和 IP 环境。

### Q: 可以联系 Anthropic 客服解锁吗？

Anthropic 没有提供针对手机号验证的客服支持渠道。临时封锁只能等待自动解除。

### Q: 换了号码但 IP 还是同一个，可以吗？

如果只是号码被锁（不是 IP），换号码可能有效。但如果 IP 也被锁了，需要同时更换 IP。
